July 13, 2022: Email Campaign ServiceFlow

The Email Campaign ServiceFlow has been added:

  • The ability to send Email Campaigns (email broadcasts, newsletters, targetted emails etc. has been added
  • Any user account in the Staff security group can create an Email Campaign.
    1. The Sender starts with a drafting the Campaign Email. They can work on this as a draft until they select "Ready to Send" as Yes.
    2. The Sender is then sent a Test Email. They have the option to Agree, Revise or Decline:
      • Agree will mark the task as accepted and task Email Campaign Reviews a Review Campaign Email task
      • Revise will re-send another test email based on the Subject and Message fields in the Email Campaign
      • Decline will end the ServiceFlow
    3. The ServiceFlow then emails the proposed Email Campaign to Email Campaign Reviewers:
      • Email Campaign Reviewers are identified in the PEePs list.
      • If a Dear Soul only wishes to be a Reviewer for a particular email, then they can be added just before the Sender Agrees with the Test Email.
      • Alternatively, if a Reviewer is perceived as missing then any Reviewer can mark the Email Campaign as "Revise", returning the ServiceFlow to the Test Email state, and then the identified new Reviewer can be added to the PEePs list.
    4. Finally, the Email Campaign Review task will have three outcome options to select from:
      • Agree will mark the task as accepted. When all Reviewers Agree then the Email Campaign is sent and the ServiceFlow marked complete
      • Revise will end the Review task for all Reviewers, and task the original Sender with a task to review the Reviewer Comments and make revisions
      • Decline will end the ServiceFlow

Additional Changes
  • Task/notification mmails now "fail silently". In other words, if the email server send returns an error this does not cause a client crash.
  • Added the Learned Experience Engine (LEE). This allows for ServiceFlow task checklists to be updated without development effort.
  • Series, Tag, and Master attributes are now unique, thus preventing duplicate entries spelled exactly the same way.
  • Added the Process Expertise Profile (PEePs) list. This allows for ServiceFlows to be programmed to assign tasks to Dear Souls correctly without development effort.
  • Added the Profiles page. This displays all Profiles in the system. Profiles will be a starting point for ServiceFlows and other Profile-centric activity. For example, clicking on a Profile displays the Reading List for each Profile along with their Profile details.
  • Added the Audience Model. Audiences allow the Email Campaign ServiceFlow to direct campaigns to targetted groups of emails.
  • Added the Mailing List Model. Mailing Lists store all user accounts or email addresses used by the Email Campaign. Each Mailing List record has one Audience. Many Dear Souls/email subscribers can belong to many Audiences. However, a Dear Soul/email subscriber can only belong to an Audience once.
  • Email Campaign Model added (refer to Email Campaign ServiceFlow above for more details).
  • Librarian Training updated with Record Observation ServiceFlow video.
  • Code clean up: Removed argument calls for Super Class; Added "related_name" attribute to all ForeignKey rows.
July 2, 2022: Library Observation ServiceFlow

The Library Observation ServiceFlow has been updated:

  • The initial observation form has been updated. Observation Types are now: Typo; Image; Broken Link; Other
  • Upon submission the ServiceFlow will (depending on the Librarian's profile notification preference*) send an email notification to all Digital Librarians:
    • *You can verify your Notification Preference on your Profile here (scroll to the bottom of the page)
    • When set to "EMail" then you will be emailed if you are in the applicable service group
    • When set to "None" you will not be notified by email
    • However, password reset emails will still work regardless of your notification preference
  • The Library Task list stores all active and completed Library tasks
  • Field descriptions for key task fields have been added
  • The Book Editor task presents all preceding task information as well as any entered urls for book files
